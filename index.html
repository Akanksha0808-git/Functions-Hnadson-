 Q4-  "Print output: 
        var x = 21;
        var girl = function () {
            console.log(x);
            var x = 20;
        };
        girl ();"

ANS= Output : undefined
Explaination : The inner x variable is defined after the console.log(x) statement, the value of the inner x variable is not accessible at that point. Therefore, when console.log(x) is executed, the value of x is undefined. 

Q5- "Print output: 
            var x = 21;
            girl ();
            console.log(x)
            function girl() {
                console.log(x);
                var x = 20;
            };
            "

ANS = Output : undefined 21
Explanation : The first console.log(x) inside the girl function outputs undefined because of hoisting, and the second console.log(x) outside the girl function outputs 21, the value of the global variable x.

Q7- Write a function that accepts parameter n and returns factorial of n

ANS= Output : 40
Explanation: In this function, we first check if n is either 0 or 1. If so, we return 1, as the factorial of 0 and 1 is 1. For any other value of n, we initialize a variable result to 1. Then, we iterate from 2 to n and multiply result by each number in the range. Finally, we return the resulting result, which will be the factorial of n.

Q2- "Guess the output

                    Abc();
                    const Abc = function(){
                        let value = 20;
                        console.log(value);
                    }"

ANS = Output: runtime error.
Explanation :The error occurs because you are trying to invoke the function Abc() before it is declared. In JavaScript, function declarations are hoisted to the top of their scope, meaning they can be called before they are declared. However, function expressions, such as the one you have used (const Abc = function() {...}), are not hoisted and cannot be called before they are defined. 

Q3- "Guess the output

                        var a = 10;
                        (function (){
                            console.log(a);
                            var a = 20;
                        })();"

ANS= Output : undefined.
Explaination : Due to hoisting, the local variable a is hoisted to the top of the function scope. Therefore, when the code executes console.log(a) before the local variable a is assigned the value 20, the local variable a is in a "temporal dead zone" and has the value undefined at that point.
